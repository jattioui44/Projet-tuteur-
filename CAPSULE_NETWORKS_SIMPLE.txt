🤔 C'EST QUOI UNE CAPSULE?
═══════════════════════════════════════════════════════════════════════════════

NEURONE CLASSIQUE (CNN)
───────────────────────

Un nombre (scalaire):
┌───────┐
│  0.8  │  ← "Il y a probablement un œil"
└───────┘

Problème:
❌ Pas d'information sur la position
❌ Pas d'information sur l'orientation
❌ Pas d'information sur la taille

CAPSULE
───────

Un vecteur (plusieurs nombres):
┌─────────────────┐
│  [0.2, 0.8, 0.1, │  ← Plusieurs informations!
│   0.5, 0.3, ...]│
└─────────────────┘

Longueur du vecteur = "Est-ce qu'il y a un œil?"
Direction du vecteur = "Où? Comment? Quelle taille?"

Avantages:
✅ Position préservée
✅ Orientation préservée
✅ Propriétés préservées

IMAGINEZ UN VISAGE
──────────────────

CNN Classique dit:
"Je vois un œil: 90% de confiance"
"Je vois un nez: 85% de confiance"
"Je vois une bouche: 80% de confiance"
→ Donc c'est un visage!

Problème:
┌─────┐
│ œil │
│ nez │ ← Mais dans le mauvais ordre! ❌
│bouche
└─────┘

Le CNN dira quand même "c'est un visage"!

Capsule Network dit:
"Je vois un œil à (x=10, y=5), orienté à 0°"
"Je vois un nez à (x=10, y=15), orienté à 0°"
"Je vois une bouche à (x=10, y=25), orienté à 0°"
→ Les positions sont correctes! ✅

┌─────┐
│ œil │
│ nez │ ← Ordre correct! ✅
│bouche│
└─────┘

🎯 CONCEPTS CLÉ
═══════════════════════════════════════════════════════════════════════════════

1. VECTEUR = PROPRIÉTÉS
───────────────────────

Chaque dimension du vecteur représente une propriété:
[position_x, position_y, rotation, taille, épaisseur, ...]

Exemple pour un vaisseau sanguin:
[0.2, 0.8, 0.1, 0.9, 0.3]
 │    │    │    │    └─── épaisseur: 0.3
 │    │    │    └──────── longueur: 0.9
 │    │    └───────────── angle: 0.1 rad
 │    └────────────────── position_y: 0.8
 └─────────────────────── position_x: 0.2

2. LONGUEUR = PROBABILITÉ
─────────────────────────

Longueur du vecteur = "Est-ce que cette entité existe?"

Vecteur court (proche de 0):
||v|| = 0.1  → "Probablement pas un vaisseau"

Vecteur long (proche de 1):
||v|| = 0.9  → "Très probablement un vaisseau!"

3. SQUASH FUNCTION
──────────────────

Pour garder la longueur entre 0 et 1:

Avant squash:
[100, 200, 50]  → Longueur = 228 (trop grand!)

Après squash:
[0.4, 0.8, 0.2]  → Longueur = 0.89 (entre 0 et 1) ✅

Comme une "normalisation douce" qui préserve la direction.

4. ROUTING BY AGREEMENT
───────────────────────

Au lieu de max pooling (prendre le maximum), les capsules
"votent" pour leur parent!

Capsules enfants:      Capsule parent:
                       "Micro-anévrisme"
  "Rond"    ────┐
                 ├─→  Si tout le monde
  "Rouge"   ────┤     est d'accord ✅
                 │     → Forte confiance!
  "Petit"   ────┘

Si une capsule n'est pas d'accord:
                       "Micro-anévrisme"
  "Rond"    ────┐
                 ├─→  Désaccord ❌
  "Vert"    ────┤     → Faible confiance
                 │     
  "Grand"   ────┘

Avantage:
✅ Apprend les bonnes relations
✅ Robuste au bruit
✅ Hiérarchies naturelles

🏗️  ARCHITECTURE CAPSULE NETWORK
═══════════════════════════════════════════════════════════════════════════════

ÉTAPES SIMPLES
──────────────

1. FEATURES CNN
   Image → Convolutions → Features
   [192, 192, 3] → [6, 6, 256]

2. PRIMARY CAPSULES
   Features → Première couche de capsules
   [6, 6, 256] → 32 capsules de 8D
   
   Rôle: Détecter entités de bas niveau
   (contours, textures, petites lésions)

3. ROUTING
   Capsules primaires → Vote pour parents
   32 capsules → 5 capsules (classes)
   
   Rôle: Apprendre les bonnes relations
   "Ces capsules forment un micro-anévrisme"

4. CLASS CAPSULES
   Une capsule par classe (5 classes)
   Longueur = probabilité de la classe
   
   Rôle: Représentation finale
   Classe 0: ||v₀|| = 0.1 → 10% de confiance
   Classe 2: ||v₂|| = 0.9 → 90% de confiance ✅

💪 POURQUOI C'EST PUISSANT?
═══════════════════════════════════════════════════════════════════════════════

POUR RÉTINOPATHIE DIABÉTIQUE
─────────────────────────────

1. RELATIONS SPATIALES
   
   Exemple: Exsudats
   CNN: "Je vois des taches jaunes"
   Capsule: "Je vois des taches jaunes en amas près
             de la macula, pattern caractéristique
             d'exsudats durs" ✅

2. HIÉRARCHIES
   
   Micro → Macro:
   Pixels rouges
      → Micro-hémorragie
         → Cluster de micro-hémorragies
            → Rétinopathie modérée

   Capsules préservent cette hiérarchie!

3. ROBUSTESSE
   
   Image tournée de 15°:
   CNN: Peut échouer ❌
   Capsule: Vecteur encode la rotation ✅
           → Robuste!

4. INTERPRÉTABILITÉ
   
   On peut voir ce que chaque capsule a appris:
   "Capsule 3 détecte des vaisseaux fins"
   "Capsule 7 détecte des néovascularisations"
   
   → Aide au diagnostic clinique!
